activating event:きっかけ　　例：携帯の通知がなった 
Belief:その時の自動思考　　例：五分だけなら大丈夫 
Consequences:結果　　例：ショーと動画に30分感浪費してしまった、罪悪感は8/10 
Disputation:反論　　例：本当に今すぐ見る必要あった？ 
Effect:新しい考え・行動　　例：通知をオフにする、帰宅したらスマホをポストにしまう

#問題

アンケートをどう取るのか 

役割分担と開発環境・ツールの整備 
友達同士だからこそ、責任の所在とコミュニケーションの場を明確にしておくことが重要です。

役割分担: UI/UXデザイン、フロントエンド実装、バックエンド（サーバー・データベース）、進行管理など、得意分野に合わせてメインの担当を決めます。

情報共有・タスク管理: TrelloやNotionなどで「誰が・いつまでに・何をするか」を可視化します。

バージョン管理: 複数人でコードを触るなら Git / GitHub の導入は必須です。これがないと、お互いのコードを上書きして消してしまう事故が起きます。

コミュニケーション: DiscordやSlackなどに開発専用のサーバー/チャンネルを作り、雑談とは分けてログを残せるようにします。
